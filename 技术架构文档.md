## 三、双模型修复策略详解

### 3.1 模型对比

| 特性 | MI-GAN | LaMa |
|------|--------|------|
| **模型大小** | ~28MB | ~200MB |
| **输入尺寸** | 任意尺寸 | 固定 512x512 |
| **处理速度** | 快 | 较慢 |
| **修复质量** | 适合小区域 | 适合大区域，效果好 |
| **使用场景** | 🔴 消除区域 | 🟣 替换区域背景消除 |

### 3.2 关键处理逻辑

#### MI-GAN 处理 (消除区域)

```python
# 基于原图分别处理，避免多区域互相影响
original_img = img.copy()
for i, bbox in enumerate(erase_bboxes):
    single_mask = make_mask(img.size, [bbox])
    # 始终基于原图处理
    result = inpaint_migan(original_img, single_mask, preserve_unmasked=True)
    # 只取mask区域的修复结果
    img = merge_inpaint_result(img, result, single_mask)
```

**特点：**
- 每个区域基于**原图**独立处理
- 使用 `preserve_unmasked=True` 保护非 mask 区域
- 避免链式处理的累积误差

#### LaMa 处理 (替换区域背景消除)

```python
for i, bbox in enumerate(replace_bboxes):
    single_mask = make_mask(img.size, [bbox])
    # 使用 LaMa 消除背景，保护其他区域
    img = inpaint_lama(img, single_mask, preserve_unmasked=True)
```

**特点：**
- LaMa 固定输入 512x512，会将图像 resize 处理
- 使用 `preserve_unmasked=True` 保护非 mask 区域
- 适合大区域的背景消除，效果更好

### 3.3 merge_inpaint_result 函数

```python
def merge_inpaint_result(base_img, inpaint_result, mask):
    """
    只取mask区域内的修复结果，其他区域保持基础图像不变
    
    mask=255 的区域: 使用修复结果
    mask=0   的区域: 保持基础图像
    """
    mask_array = np.array(mask.convert('L'))
    base_array = np.array(base_img)
    result_array = np.array(inpaint_result)
    
    # mask=255的区域使用修复结果，mask=0的区域保持基础图像
    mask_3ch = np.stack([mask_array] * 3, axis=-1) / 255.0
    combined = (result_array * mask_3ch + base_array * (1 - mask_3ch)).astype(np.uint8)
    
    return Image.fromarray(combined, mode='RGB')
```

### 5.1 主体识别 (detect_subject_auto)

```
输入图片
    │
    ├── GrabCut 算法 ───────┐
    ├── 显著性检测 ─────────┼──▶ 综合权重图 ──▶ 二值化
    ├── 边缘检测 ───────────┤
    ├── 颜色聚类 ───────────┘
    └── 中心偏好 (主体通常在中心)
```

### 5.2 抠图处理流程

```
主体抠图
    │
    ├── 提取主体区域 (crop)
    ├── 创建 mask (主体区域为白色)
    ├── 背景处理:
    │   ├── transparent: 透明背景
    │   ├── white: 白色背景
    │   ├── blur: 高斯模糊背景
    │   └── inpaint: 智能填充背景
    └── 贴回主体
```

## 八、关键设计决策

### 8.1 为什么消除和替换使用不同模型？

| 功能 | 使用模型 | 原因 |
|------|----------|------|
| 🔴 消除 | MI-GAN | 小模型，速度快，支持任意尺寸，适合小区域快速消除 |
| 🟣 替换背景消除 | LaMa | 大模型，修复效果好，适合需要高质量背景填充的场景 |

### 8.2 为什么每个区域要基于原图处理？

**问题：** 链式处理会导致累积误差，多个消除区域互相影响

**解决方案：**
```
原图 ─→ 修复区域A（基于原图）→ 结果A ─┐
   ├→ 修复区域B（基于原图）→ 结果B ─┼→ 合并
   └→ 修复区域C（基于原图）→ 结果C ─┘
```

每个区域独立基于原图修复，避免互相影响。

### 8.3 为什么 LaMa 需要 preserve_unmasked？

**问题：** LaMa 固定输入 512x512，会将整张图 resize 处理，导致非 mask 区域像素变化

**解决方案：** 使用 `merge_inpaint_result` 只取 mask 区域的修复结果，其他区域保持原图不变
