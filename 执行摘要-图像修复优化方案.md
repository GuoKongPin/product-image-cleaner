# 执行摘要：商品图片清洗工具 - 图像修复优化方案

## 一、背景与问题

### 1.1 技术选型

本项目采用 **MI-GAN** 和 **LaMa** 两种主流的 ONNX 图像修复模型，分别用于不同的处理场景：

| 模型 | 文件 | 大小 | 特点 | 适用场景 |
|------|------|------|------|----------|
| **MI-GAN** | `migan_pipeline_v2.onnx` | ~28MB | 轻量、快速、任意尺寸输入 | 🔴 消除区域（背景填充） |
| **LaMa** | `lama_fp32.onnx` | ~200MB | 高质量、大感受野、固定512×512输入 | 🟣 替换区域（背景消除+文字替换） |

### 1.2 核心问题

当用户提供**多个边界框**（作为掩码）时，传统的逐个处理方式面临以下挑战：

1. **掩码质量下降** - 每个掩码范围小、上下文信息不足
2. **边界融合不自然** - 每次修复后边缘生硬，产生拼接痕迹
3. **一致性难以保证** - 独立修复可能产生视觉不连贯
4. **性能开销大** - 每个框都需要预处理、前向推理、后处理

## 二、模型架构与技术原理

### 2.1 MI-GAN（Mobile Inpainting GAN）

**核心创新点：**

| 技术 | 说明 | 效果 |
|------|------|------|
| **深度可分离卷积** | 将标准卷积分解为 depthwise + pointwise | 减少参数量和计算量 |
| **重参数化技巧** | 训练时可拆分结构，推理时等价合并 | 加速推理速度 |
| **知识蒸馏** | 使用 Co-Mod-GAN（StyleGAN-v2）作为教师网络 | 小模型达到大模型质量 |
| **双分支结构** | 主分支（U-Net）+ 绘制分支（多步细节填充） | 从整体结构到细节逐层修复 |

**输入输出：**
- 输入：RGB图像（uint8）+ 二值掩码（uint8，0=遮盖区域，255=已知区域）
- 输出：修复后的RGB图像，形状与原图相同
- 预处理：自动归一化、裁剪、缩放到512×512
- 后处理：反归一化、双线性插值回原尺寸、融合到原图

**输出公式：**
```
I_out = I_orig ⊙ M + I_c ⊙ (1 - M)
```
其中 `I_orig` 为原图，`I_c` 为网络生成结果，`M` 为掩码。

### 2.2 LaMa（Large Mask Inpainting）

**核心创新点：**

| 技术 | 说明 | 效果 |
|------|------|------|
| **傅里叶卷积（FFC）** | 部分通道在频域执行卷积操作 | 获得极高的感受野，适合大孔洞修复 |
| **ResNet-like 结构** | 3个下采样块 + 6-18个残差块 + 3个上采样块 | 强大的特征提取能力 |
| **高感受野感知损失** | High Receptive Field Perceptual Loss | 保证全局语义和频域一致性 |

**损失函数组合：**
```
L_total = L_Adv + L_HRFPL + L_DiscPL + 梯度惩罚
```

**输入输出：**
- 输入：固定512×512尺寸（图像和掩码都会resize）
- 输出：512×512修复结果，再插值回原尺寸
- 特点：适合大面积遮罩，修复质量高但速度较慢

## 三、优化策略与实现

### 3.1 策略一：智能合并邻近Mask

**问题：** 多个邻近小区域分别处理，产生多次推理开销和边界痕迹

**解决方案：**

```python
def merge_nearby_bboxes(bboxes, image_size, merge_distance=30):
    """
    1. 将每个bbox转为二值mask
    2. 形态学膨胀（椭圆核，半径30px）连接邻近区域
    3. 闭运算填充小空洞
    4. 提取连通域作为新的合并区域
    """
```

**触发条件：**
- 区域数量 ≥ 3个
- 总消除面积占比 ≤ 30%

**效果对比：**

| 场景 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 5个邻近小区域 | MI-GAN ×5 = 1.0s | 合并 + LaMa ×1 = 1.5s | 质量↑，纹理连续 |
| 营养成分表 | 多次推理，边界生硬 | 统一修复，全局一致 | 视觉效果↑ |

### 3.2 策略二：智能模型选择

**决策逻辑：**

```
消除区域：
    ├── 满足合并条件 ──▶ 合并 + LaMa 统一修复
    └── 不满足合并条件 ──▶ MI-GAN 逐个处理（基于原图）

画笔涂抹：
    ├── 面积占比 > 5% ──▶ LaMa
    └── 面积占比 ≤ 5% ──▶ MI-GAN
```

**配置参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `LAMA_AREA_THRESHOLD` | 0.05 | LaMa触发阈值（5%） |
| `MERGE_MASKS_ENABLED` | true | 是否开启合并 |
| `MERGE_MASKS_DISTANCE` | 30 | 合并距离阈值（像素） |
| `MERGE_MASKS_MIN_AREAS` | 3 | 最少区域数触发合并 |

### 3.3 策略三：Mask边缘羽化

**问题：** 硬边缘拼接产生明显边界痕迹

**解决方案：**

```python
def merge_inpaint_result(base_img, inpaint_result, mask,
```

### 3.4 策略四：替换区域批量处理

**优化前：**
```python
# 逐个处理，多次LaMa推理
for bbox in replace_bboxes:
    mask = make_mask([bbox])
    img = inpaint_lama(img, mask)  # N次推理
```

**优化后：**
```python
# 批量合并，一次性消除所有背景
if len(replace_bboxes) > 1:
    combined_mask = make_mask(img.size, replace_bboxes)
    img = inpaint_lama(img, combined_mask)  # 1次推理
```

**性能对比：**

| 场景 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 3个替换区域 | LaMa ×3 = 4.5s | LaMa ×1 = 1.5s | **3×速度↑** |
| 5个替换区域 | LaMa ×5 = 7.5s | LaMa ×1 = 1.5s | **5×速度↑** |

### 3.5 策略五：基于原图的增量处理

**问题：** 链式处理导致累积误差，多个区域互相影响

**解决方案：**

```python
# 保存原图
original_img = img.copy()

# 每个区域基于原图独立处理
for bbox in erase_bboxes:
    result = inpaint_migan(original_img, single_mask)
    # 只取mask区域合并到当前结果
    img = merge_inpaint_result(img, result, single_mask)
```

**处理流程：**

```
原图 ─→ 修复区域A（基于原图）→ 结果A ─┐
   ├→ 修复区域B（基于原图）→ 结果B ─┼→ 合并
   └→ 修复区域C（基于原图）→ 结果C ─┘
```

## 五、配置与使用

### 5.1 环境变量配置

```bash
# 模型选择
export INPAINT_MODEL=auto              # auto / migan / lama
export LAMA_AREA_THRESHOLD=0.05        # 5%，超过则用LaMa

# 合并相关
export MERGE_MASKS_ENABLED=true        # 是否开启合并
export MERGE_MASKS_DISTANCE=30         # 合并距离阈值（像素）
export MERGE_MASKS_MIN_AREAS=3         # 最少区域数触发合并
export MERGE_MASKS_MAX_TOTAL_RATIO=0.30 # 最大合并区域占比

# 羽化相关
export FEATHERING_ENABLED=true         # 是否开启羽化
export FEATHERING_RADIUS=3             # 羽化半径（像素）
```

### 5.2 场景推荐配置

| 场景 | 推荐配置 | 说明 |
|------|----------|------|
| 营养成分表/配料表 | `MERGE_MASKS_DISTANCE=40` | 文字行间距较大，需要更大的合并距离 |
| 分散的小logo | `MERGE_MASKS_DISTANCE=20` | 避免过度合并，保持分散处理 |
| 高质量要求 | `FEATHERING_RADIUS=5` | 更强的羽化效果，消除边界痕迹 |
| 快速处理 | `MERGE_MASKS_ENABLED=false` | 跳过合并逻辑，直接逐个处理 |

## 八、总结

本优化方案针对商品图片清洗工具的消除与替换功能，实现了以下核心改进：

1. **智能合并** - 将邻近小区域合并为大区域，减少推理次数，提升纹理连续性
2. **智能模型选择** - 根据区域面积和分布自动选择最优模型组合
3. **边缘羽化** - 消除拼接痕迹，实现自然过渡
4. **批量处理** - 替换区域批量合并，大幅提升处理速度
5. **基于原图处理** - 避免链式累积误差，保证多区域处理质量

通过这些优化，系统在保持高质量修复效果的同时，显著提升了处理效率和用户体验。
